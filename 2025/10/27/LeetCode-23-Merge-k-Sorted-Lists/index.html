LeetCode 23 - Merge k Sorted Lists

LeetCode 23 - Merge k Sorted Lists

👉 題目連結

📖題目說明

You are given an array of k linked-lists lists, each linked-list is sorted in ascending order.
Merge all the linked-lists into one sorted linked-list and return it.

給你一個 k 個鏈結串列的陣列 lists，每個鏈結串列都已按升序排列
請你將所有鏈結串列合併成一個升序排列的鏈結串列，並返回合併後的鏈結串列

範例輸入與輸出

Example 1

Input: lists = [[1,4,5],[1,3,4],[2,6]]
Output: [1,1,2,3,4,4,5,6]
Explanation: The linked-lists are:
[
1->4->5,
1->3->4,
2->6
]
merging them into one sorted linked list:
1->1->2->3->4->4->5->6

Example 2

Input: lists = []
Output: []

Example 3

Input: lists = [[]]
Output: []

Constraints

  • k == lists.length
  • 0 <= k <= 10^4
  • 0 <= lists[i].length <= 500
  • -10^4 <= lists[i][j] <= 10^4
  • list[i]按升序排序
  • list[i].length 的總和不會超過 10^4

Solution

逐一合併

解題想法:
這是一個最直觀的延伸作法，既然我會合併 2 個串列（LeetCode 21），那我只要重複這個操作就行了
程式碼中，我先以第一個元素為基礎 (merged)，然後用一個 for 迴圈，依序將 lists[1], lists[2], … 逐一呼叫 self.mergeTwoLists 合併到 merged 上
這個方法雖然簡單，但效率很差，因為 merged 串列會越長，導致後面的合併操作越來越耗時

時間複雜度: O(N．k)，其中 N 是總節點數，k 是串列數，每次合併的成本與 N 相關，總共要合併 k 次
空間複雜度: O(1)，只使用了幾個額外的指標，所以是常數空間

分治法

解題想法:
逐一合併法太慢，我的想法是使用分治思法優化這個過程
與其 (L1+L2)+L3 這樣合併，不如 (L1+L2) 和 (L3+L4) 先各自合併，再把結果合起來
程式碼中的 merge函式不斷地將 k 個串列的陣列對半切分，直到left == right時返回
然後，它會取得左右兩半各自遞迴合併的結果，最後再呼叫 mergeTwoLists 將這兩個大小相近的結果串列合併

時間複雜度: O(N logk)，總共有 N 個節點，透過分治法，每個節點都參與了 log k 層的合併
空間複雜度: O(logk)，空間開銷主要來自於遞迴呼叫的堆疊深度

最小堆

解題想法:
這個方法的思路是，在任何時刻，合併後串列的下一個節點必定是 k 個串列當前頭節點中最小的那一個
如何高效地在 k 個元素中找到最小值？答案就是最小堆（Min-Heap）

  1. 建立一個虛擬節點 dummy 和一個最小堆 min_heap
  2. 將k個串列的頭節點全部放入堆中，程式碼中 (l.val, counter, l) 是一個技巧，heapq 會根據元組的第一個元素 l.val 來排序，counter 則用來在值相同時比較，避免比較 ListNode 物件本身而出現錯誤
  3. 迴圈開始，heappop 會彈出 k 個候選者中最小的那個 node
  4. 我將這個 node 串接到 current.next
  5. 將剛彈出節點的下一個節點node.next 重新 heappush 推入堆中，作為新的候選者參與下一輪的比較

時間複雜度: O(N log k)，總共有N個節點，堆的大小最多為 k
空間複雜度: O(k)，空間開銷來自於最小堆，堆中最多會儲存 k 個節點
