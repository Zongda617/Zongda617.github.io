LeetCode 21 - Merge Two Sorted Lists

LeetCode 21 - Merge Two Sorted Lists

👉 題目連結

📖題目說明

You are given the heads of two sorted linked lists list1 and list2.
Merge the two lists into one sorted list. The list should be made by splicing together the nodes of the first two lists.
Return the head of the merged linked list.

給兩個已排序linked lists list1 和 list2 的頭
將這兩個linked lists合併為一個已排序列表
此清單應由前兩個linked lists的節點拼接而成
傳回合併後linked lists的頭

範例輸入與輸出

Example 1

Input: list1 = [1,2,4], list2 = [1,3,4]
Output: [1,1,2,3,4,4]

Example 2

Input: list1 = [], list2 = []
Output: []

Example 3

Input: list1 = [], list2 = [0]
Output: [0]

Constraints

  • 兩個linked lists中的節點數都在 [0, 50] 範圍內
  • list1 和 list2 均依非降序排序
  • -100 <= Node.val <= 100

Solution

迭代法

解題想法:
創建一個新的鏈結串列，然後逐一比較兩個原始串列的節點，將較小的節點串接到新的串列後面

  1. 設立虛擬節點 dummy：這個節點本身不儲存任何有效值，它的主要目的是作為新串列的假頭，這樣我們就不需要處理頭節點為空的邊界情況
  2. 移動的當前指標：再創建一個 current 指標，初始指向 dummy，這個指標負責在新串列上移動，並將下一個節點串接上來
  3. 比較與串接：
    • 當 list1 和 list2 都還沒遍歷完時，比較它們當前節點的值
    • 將值較小的那個節點接到 current.next
    • 移動被選中那個串列的指標
    • 移動 current 指標到剛剛接上的新節點上
  4. 處理剩餘部分：當迴圈結束時，list1 或 list2 其中一個可能還有剩餘的節點，因為它們本身已經是排序好的，所以我們直接將整個剩餘的串列接到 current.next

時間複雜度: $O(m + n)$，其中 m 和 n 分別是兩個鏈結串列的長度
空間複雜度: $O(1)$，只有使用額外的指標，所以是常數空間

遞迴法

解題想法:
遞迴的核心思想是：merge的結果，取決於哪個的值更小

  1. 終止條件：
    如果其中一個串列是空的就不需合併，直接回返另一個串列
  2. 遞迴步驟：
    比較兩個串列當前節點的值
    如果 list1.val 比較小，那麼合併後的新串列的頭節點就是 list1
    接下來的問題就變成了合併 list1.next 和 list2，所以，我們讓 list1.next 指向 merge(list1.next, list2) 的返回結果，最後返回 list1
    相反也是如此
    時間複雜度: $O(M + N)$，其中 m 和 n 分別是兩個鏈結串列的長度
    空間複雜度: $O(M + N)$，主要來自於遞迴呼叫的堆疊深度
