LeetCode 14 - Longest Common Prefix

LeetCode 14 - Longest Common Prefix

👉 題目連結

📖題目說明

Write a function to find the longest common prefix string amongst an array of strings.
If there is no common prefix, return an empty string "".

尋找字串陣列中最長的公共前綴字串，如果沒有公共前綴，則傳回空字串""

範例輸入與輸出

Example 1

Input: strs = [“flower”,”flow”,”flight”]
Output: “fl”

Example 2

Input: strs = [“dog”,”racecar”,”car”]
Output: “”
Explanation: There is no common prefix among the input strings.

Constraints

  • 1 <= strs.length <= 200
  • 0 <= strs[i].length <= 200
  • strs[i] 由小寫英文字母組成

Solution

水平掃描

解題想法:
這個方法的核心概念是逐一縮短前綴
先假設整個陣列的第一個字串是目前的共同前綴，接著從第二個字串開始逐一比對
只要發現當前字串不是以這個前綴開頭，就不斷把前綴從尾端削短，直到它變成當前字串的前綴為止
這樣每次比對都會讓 prefix 越來越短，效率反而很好
個人覺得這個方法是最直觀且實務上最穩定的寫法，在大多數情況下效能都非常接近最佳

時間複雜度: O(n) , 其中 n 為所有字元的總數
空間複雜度: O(1) , 只需常數額外空間

垂直掃描

解題想法:
垂直掃描法是從字元位置的角度來思考
把第一個字串當作基準，然後從第 0 個字元開始，檢查所有字串在這個位置是否相同
若有任意字串在該位置結束或字元不同，就代表共同前綴結束
這個方法比水平掃描更早停止
而且思路很直觀就像逐欄檢查一個文字表格
實際上它和水平掃描在時間複雜度上相同，屬於最簡潔且易於理解的寫法之一
時間複雜度: O(n) , 其中 n 為所有字元的總數
空間複雜度: O(1) , 只需常數額外空間

二分搜尋

解題想法:
這個方法是從前綴長度的角度出發，而不是直接比字串
我會先找到最短字串的長度 Lmin，然後對 [0, Lmin] 進行二分搜尋
每次取中間長度 mid，檢查所有字串是否都包含該長度的共同前綴
若是，就嘗試更長，否則縮短
這樣能在某些情況下減少重複比對，尤其是字串很長但差異發生在後段時
不過實務上它比 O(n) 的掃描法多一個 log(L) 的開銷

時間複雜度: O(n · log Lmin)，其中 Lmin 為最短字串長度
空間複雜度: O(1)

分治法

解題想法:
把字串陣列分成左右兩半，分別遞迴求出各自的最長共同前綴
最後再合併兩邊的結果
這樣做的好處是概念清楚，而且在理論上能以遞迴的方式自然分解問題
不過實務上，因為每層都要比較前綴、並維護遞迴堆疊，實際速度未必比單純掃描快
但我覺得這個方法非常適合用來展示演算法設計思維
尤其是在如何拆問題和如何合結果的教學中很有代表性

時間複雜度: O(n) , n 為所有字元的總數
空間複雜度: O(m · log n) , m 為最長共同前綴長度，n 為字串數
