LeetCode 22 - Generate Parentheses

LeetCode 22 - Generate Parentheses

👉 題目連結

📖題目說明

Given n pairs of parentheses, write a function to generate all combinations of well-formed parentheses.

給一個整數 n ，找出所有 n 對括號的有效組合

範例輸入與輸出

Example 1

Input: n = 3
Output: [“((()))”,”(()())”,”(())()”,”()(())”,”()()()”]

Example 2

Input: n = 1
Output: [“()”]

Constraints

  • 1 <= n <= 8

Solution

回溯法

解題想法:
我的想法是透過回溯法來建構所有可能的括號組合
我使用一個 backtracking 函式來追蹤當前已使用的左括號和右括號數量
在每一步，我都嘗試做出選擇：放左括號( 或右括號 )

  • 放左括號：只要已用的左括號 open 還沒達到 n ，我就可以安全地放一個 (
  • 放右括號：只要已用的右括號 close 數量小於左括號 open ，我才能放一個 ) 來配對
    當 open == n 且 close == n 時，表示一個完整的組合建構完畢，就將它加入 result
    整個過程的關鍵在於 path.pop()，這就是回溯的精隨
    當一條路徑探索完畢並返回後，我必須 path.pop() 來撤銷上一步的 (
    這樣程式才能回到上一個岔路口去嘗試另一條分支，也就是放)
    這個選擇 → 遞迴 → 撤銷的循環保證了所有路徑都會被不重複地探索

時間複雜度: O(4^n / √n)，這是一個與卡特蘭數相關的複雜度，組合的總數是卡特蘭數 C_n，而每個組合需要 O(n) 的時間來建構，總體時間複雜度大致是 O(n．C_n)
空間複雜度: O(n)，遞迴呼叫的深度

動態規劃

解題想法:
運用動態規劃將大問題拆解成小問題，我的想法是，n=3 的答案，一定可以由 n=0, n=1, n=2 的答案拼湊而成
這個問題的一個關鍵結構：任何一個 n > 0 的有效括號組合，都可以被唯一地拆解成 (A)B 的形式

  • ( 和 ) 是最外層的第一對括號
  • A 是被這對括號包在裡面的有效組合（可能為空）
  • B 是在這對括號後面的有效組合（可能為空）
    如果我們總共需要 n 對括號，那麼 (A)B 這個結構已經用掉了 1 對，剩下的 n-1 對括號，會被分配給 A 和 B
    假設 A 用了 i 對括號，B 用了 j 對括號，那麼 i + j = n - 1
    這就啟發了我使用 DP：
  1. 狀態定義：dp[k] 是一個列表，儲存了 k 對括號能組成的所有有效組合
  2. 初始值：dp[0] = [""] (0 對括號只有空字串一種組合)
  3. 狀態轉移：要計算 dp[k]，我就去遍歷 i 從 0 到 k-1 的所有可能
    • 令 j = k - 1 - i
    • 我去 dp[i] 裡取出所有 A 的組合
    • 我去 dp[j] 裡取出所有 B 的組合
    • 然後用雙層迴圈把它們拼裝成 f"({a}){b}"，其中 a 來自 dp[i]，b 來自 dp[j]
      程式碼中我使用了遞迴 + 記憶化的方式來實現

時間複雜度: O(4^n / √n)，與回溯法相同，都受卡特蘭數的增長速度所限制
空間複雜度: O(4^n / √n)，這是 DP 解法的主要缺點，因為我們必須在 memo中儲存從 dp[0] 到 dp[n-1] 所有的中間結果，而這些列表的大小總和會達到與卡特蘭數同一個量級，空間開銷遠大於回溯法
