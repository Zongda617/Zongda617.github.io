LeetCode 10 - Regular Expression Matching

LeetCode 10 - Regular Expression Matching

👉 題目連結

📖題目說明

Given an input string s and a pattern p, implement regular expression matching with support for '.' and '*' where:

  • '.' Matches any single character.​​​​
  • '*' Matches zero or more of the preceding element.
    The matching should cover the entire input string (not partial).
    給定一個字串 s 和一個模式 p，請實作一個函數來判斷 s 是否完全匹配 p。
    這裡 p 的匹配規則是：
  • . ：可以匹配 任意一個字元
  • * ：表示它前面的那個字元可以 出現零次或多次

範例輸入與輸出

Example 1

Input: s = “aa”, p = “a”
Output: false
Explanation: “a” does not match the entire string “aa”.

Example 2

Input: s = “aa”, p = “a*”
Output: true
Explanation: ‘*’ means zero or more of the preceding element, ‘a’. Therefore, by repeating ‘a’ once, it becomes “aa”.

Example 3

Input: s = “ab”, p = “.“
Output: true
Explanation: “.“ means “zero or more (*) of any character (.)”.

Constraints

  • 1 <= s.length <= 20
  • 1 <= p.length <= 20
  • s 僅包含小寫英文字母
  • p 只包含小寫英文字母、.和*
  • 保證每次出現字元*時，都會有前一個有效字元進行比對

Solution

遞迴＋記憶化

解題想法:
我用遞迴來拆問題，讓 match(i, j) 表示 s[i:] 和 p[j:] 是否能對上

  1. 如果 p 走到尾巴，就看字串s是不是也剛好走完
  2. 判斷當前字元是否能匹配（同字元或 .）
  3. 如果下一個字元是 *，就有兩種可能：
    • 0次: 跳過 x*
    • 1次以上: 繼續配對 s 的字元（前提是當前字元能匹配）
  4. 用一個 memo 把已經算過的狀態存起來，避免重複計算
    個人感覺這方法很直覺，因為跟題目敘述幾乎一樣，但要小心避免爆掉，記憶化是關鍵

時間複雜度: O(m·n)
空間複雜度: O(m·n)

動態規劃 Bottom-Up

解題想法:
換成 DP 表格來做，定義 dp[i][j] 表示 s[:i] 和 p[:j] 是否能匹配

  1. 初始化 dp[0][0] = True（空字串s對空 p）
  2. 如果 p 有 x* 這種結構，可以讓空字串匹配成功，預先處理好
  3. 遍歷字串和 pattern：
  • 如果當前 p 是字元或 .，就看左上角 dp[i-1][j-1]
  • 如果當前是 *，就有兩種情況：
    • 0 次 ⇒ 看 dp[i][j-2]
    • ≥1 次 ⇒ 當 s[i-1] 能和 p[j-2] 對上時，看 dp[i-1][j]
      這個方法比遞迴更工整，表格推下來就能拿到答案

時間複雜度: O(m·n)
空間複雜度: O(m·n)
