馬拉車演算法(Manacher's Algorithm)

馬拉車演算法(Manacher’s Algorithm)

問題定義

以LeetCode 5為例子，須找到最長回文子字串

  • 回文：就是正著讀和反著讀一樣的字串
    例子：
    • "aba" 是回文
    • "abba" 也是回文
    • "abc" 就不是
  • 子字串：連續的一段字。
    例子： "abcba" 的子字串有 "a"、"bc"、"bcb" …
    所以我們的任務就是：
    在 "babad" 裡，找到最長的回文子字串，答案可能是 "bab" 或 "aba"。
    注意: 一個字 "a" 也算是回文字串

Manacher’s Algorithm

專門拿來解最長的回文子字串的最快方法，只要 **O(n)**就可以找到

詳細說明

  1. 插入特殊符號
    在處理回文時，有兩種情況：
  • 奇數長度回文：例如 "aba"，中心是 b
  • 偶數長度回文：例如 "abba"，中心在兩個 b 之間
    如果我們直接處理，還要分奇偶，很麻煩
    解法：在每個字元間插入#，頭尾也加上，(不一定要是# 只要插入的是同個符號，且符號不存在於原字串即可)
  • "abba" → #a#b#b#a#
  • "aba" → #a#b#a#
    這樣所有回文都變成「奇數長度」，我們就不用再區分兩種情況
  1. 核心概念：半徑陣列 P
    定義一個陣列 P：
  • P[i] = 以位置 i 為中心的回文半徑
    例如字串 #a#b#a#

在索引 3（字母 b），回文是 #a#b#a#，長度 7，半徑是 3
所以 P[3] = 3

3.利用對稱性

假設目前已知一個 最大回文的右邊界 right，它的中心是 center
對於一個新位置 i，我們可以找到它的「鏡像位置」：

如圖中 center 在 index = 5 的位置，也就是以 index = 5 為中心的對稱
最長的回文半徑為 3 所以 P[5] = 3
而最長的回文的右邊界就是在 index=8的位置(5+3=8)

現在有一個位置 i ，他的鏡像位置為 index=3(因為 mirror = 2 * 5 - 7)
由前面算可知 P[3]=3，所以回文字串為0-6(如圖中粉色區塊)
很明顯可以發現已經超過原本以index = 5為center的回文字串左邊界
因此最多只可以採納到i到right的距離，如以下公式

P[i] 至少可以等於 P[mirror]（但不能超過右邊界 right）

這樣做的好處是:
省掉很多不必要的比對，因為我們可以直接借用已經算好的結果
或許你還是覺得在工三小，沒關係下面跑一次流程給你看

運作流程

我們以字串 "abaaba" 為例
轉換後：

初始化：

  • center = 0
  • right = 0
  • P = [0] * 13

Step 1: i = 1 (a)

i 在 right 之外，只能自己擴展
左右比對：# 和 # 相等 → P[1] = 1
更新 center = 1, right = 2

Step 2: i = 2 (#)

i 在 right 之外
擴展後發現沒有更長 → P[2] = 0

Step 3: i = 3 (b)

i < right → 找到鏡像 mirror = -1，但不在範圍 → 自己擴展
向外比對：能擴到 #a#b#a# → P[3] = 3
更新 center = 3, right = 6

Step 4: i = 4 (#)

在右邊界內
mirror = 2*3-4 = 2 → P[2] = 0
所以 P[4] = min(right-i, P[mirror]) = min(2, 0) = 0
嘗試擴展，沒更大

Step 5: i = 5 (a)

在右邊界內
mirror = 1，P[1] = 1
所以 P[5] = min(1, 1) = 1
再嘗試擴展，發現能往外繼續延伸 → P[5] = 5
更新 center = 5, right = 10

Step 6: i = 6 (#)

在右邊界內
mirror = 4，P[4] = 0
所以 P[6] = min(4, 0) = 0
嘗試擴展 → 還能往外，得到 P[6] = 2

程式碼加註解（Python）

複雜度分析

每個位置最多只會被擴展一次 → O(n)
空間複雜度：O(n)
相比中心擴展法的 O(n²)，馬拉車演算法的效率大幅提升

個人小結

寫到這裡，我想說明:馬拉車演算法其實不是面試時必考的演算法，因為太冷門。但如果你能寫出來，絕對會讓面試官眼睛一亮
我自己在第一次學這演算法時，也覺得「哇，怎麼這麼難理解」
但當我真正理解「對稱性 + 右邊界」這個巧妙設計後，真的會有種「原來還可以這樣算！」的爽快感
所以就算你覺得程式碼不好背，至少理解它的思路，也是一個很棒的收穫
