LeetCode 5 - Longest Palindromic Substring

LeetCode 5 - Longest Palindromic Substring

👉 題目連結

📖題目說明

Given a string s, return the longest palindromic substring in s.
給定一個字串 s，傳回 s 中最長的回文子字串。
迴文是指正著念和反著念都相同，例如: “abcba”,”ravcvar”

範例輸入與輸出

Example 1

Input: s = “babad”
Output: “bab”
Explanation: “aba” is also a valid answer.

Example 2

Input: s = “cbbd”
Output: “bb”

Constraints

  • 1 <= s.length <= 1000
  • s 僅由數字和英文字母組成

Solution

暴力解

解題想法: 直接遍歷所有子字串並判斷是否為回文
時間複雜度: O(n^3)
空間複雜度: O(n)

動態規劃（DP）

解題想法: 動態規劃的關鍵在於利用「回文可以往外擴張」的特性，從短到長判斷是否為回文字串，如果子串 s[i+1..j-1] 已經是回文，而且兩端字元 s[i] 和 s[j] 相同，那麼整段 s[i..j] 也是回文，於是我們用 dp[i][j] 表示子串是否為回文，先把長度 1 與 2 的情況當作基底，再依序擴展到更長的子串，計算時按照長度由小到大，確保 dp[i+1][j-1] 先算好，過程中持續更新最長回文的位置與長度，最後得到答案
時間複雜度: O(n²)
空間複雜度: O(n²)

中心擴展

解題想法: 每個回文都有一個「中心」，可能在某個字元上（奇長）或兩個字元之間（偶長），對於字串中每個位置，我們各自把它當成這兩種中心，從中心同時向左、向右比對字元，只要還相等就繼續擴張，一旦越界或左右不同就停止，當下覆蓋到的區間就是以該中心能形成的最長回文字串
時間複雜度: O(n²)
空間複雜度: O(1)

馬拉車演算法（Manacher’s algorithm）

解題想法: 把奇數與偶數回文統一處理，先在字元之間插入分隔符（如 #）形成新字串，然後一路從左掃到右，同時維護目前「最右邊能覆蓋到的回文」及其中心（right,center），並為每個位置記一個半徑 P[i] 表示以它為中心能向外對稱匹配多遠，當走到位置 i 時，先利用它相對於 center 的鏡像點的半徑，直接得到一個不會超過 right 的最低半徑起點，只有在可能超出 right 的邊界時才繼續真正比對擴張，若擴得更右就把 center、right 往前推，因為大部分位置都能靠鏡像省掉重複比較，每個字元最多被比較兩次，整體就變成線性時間，最後從 P 的最大半徑反推回原字串的位置，即是最長回文。
時間複雜度: O(n)
空間複雜度: O(n)
