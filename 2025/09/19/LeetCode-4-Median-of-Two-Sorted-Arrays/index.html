LeetCode 4 - Median of Two Sorted Arrays

LeetCode 4 - Median of Two Sorted Arrays

👉 題目連結

📖題目說明

Given two sorted arrays nums1 and nums2 of size m and n respectively, return the median of the two sorted arrays.
The overall run time complexity should be O(log (m+n)).

給定兩個已排序陣列nums1、nums2，其長度分別為m、n，回傳兩陣列中位數
時間複雜度必須為O(log(m+n))

範例輸入與輸出

Example 1

Input: nums1 = [1,3], nums2 = [2]
Output: 2.00000
Explanation: merged array = [1,2,3] and median is 2.

Example 2

Input: nums1 = [1,2], nums2 = [3,4]
Output: 2.50000
Explanation: merged array = [1,2,3,4] and median is (2 + 3) / 2 = 2.5.

Constraints

  • nums1.length == m
  • nums2.length == n
  • 0 <= m <= 1000
  • 0 <= n <= 1000
  • 1 <= m + n <= 2000
  • $-10^6$ <= nums1[i], nums2[i] <= $10^6$

Solution

暴力解

解題想法: 直接將兩陣列合起來找中位數
時間複雜度: O((m+n) log(m+n))
空間複雜度: O(m + n)

二分搜尋法(Binary Search)

解題想法: 我們不用真的把兩個陣列合併排序，只要找到切點就能決定中位數
因為兩個陣列本來就有序，我們可以在其中一個陣列用 二元搜尋，決定切點 i，另一個陣列的切點 j 就能算出來，確保左半邊總共有一半的元素
接著檢查：左半最大 ≤ 右半最小，如果成立，表示切點正確
若總長度是奇數，中位數就是左半邊的最大值
若總長度是偶數，中位數就是左半邊最大值與右半邊最小值的平均
edge case:
當切點在陣列最左邊時，左半最大不存在 → 視為 Inifinity；
當切點在陣列最右邊時，右半最小不存在 → 視為 -Inifinity；
如果一個陣列為空，結果直接由另一個陣列決定。
時間複雜度: O(log(min(m,n)))
空間複雜度: O(1)
